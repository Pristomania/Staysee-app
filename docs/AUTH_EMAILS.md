# Брендированные письма Supabase Auth (StaySee AI)

По умолчанию Supabase отправляет стандартные письма на английском от своего SMTP. Чтобы письма были **на русском**, в стиле StaySee, и (опционально) с вашим адресом отправителя — настройте шаблоны и SMTP в проекте.

## 1. Шаблоны (тема + HTML)

Готовые файлы в репозитории:

| Шаблон | Файл | Тема письма |
|--------|------|-------------|
| Сброс пароля | `supabase/templates/recovery.html` | `Сброс пароля — StaySee AI` |
| Подтверждение регистрации | `supabase/templates/confirmation.html` | `Добро пожаловать в StaySee AI` |

### Продакшен (Dashboard) — обязательно для брендированных писем

Файлы `supabase/templates/*.html` в репозитории **не отправляются сами** на hosted Supabase. Пока шаблон не вставлен в Dashboard, письмо будет **стандартным от Supabase** (английский / без StaySee).

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard) → проект → **Authentication** → **Email Templates**.
2. **Reset password** — вставьте HTML из `recovery.html`, тему: `Сброс пароля — StaySee AI`.
3. **Confirm signup** (если включено подтверждение email) — HTML из `confirmation.html`, тема: `Добро пожаловать в StaySee AI`.
4. Нажмите **Save** на каждом шаблоне.
5. Отправьте тестовое письмо («Забыли пароль?») и убедитесь, что в письме русский текст и кнопка StaySee.

Если включён **Custom SMTP** с ошибкой — письмо может не уйти или уйти без вашего HTML; см. раздел про SMTP ниже.

### Локальная разработка

В `supabase/config.toml` уже указаны `content_path` для recovery и confirmation. После изменения шаблонов перезапустите локальный Supabase.

## 2. URL после клика по ссылке

В **Authentication** → **URL Configuration**:

- **Site URL** — ваш прод-домен (например `https://ваш-домен.ru`).
- **Redirect URLs** — добавьте все адреса, с которых открывается приложение:
  - `http://localhost:5173`
  - прод-домен (тот же, что Site URL)

Приложение обрабатывает ссылки из писем по параметру `type=` в URL:

| `type` в ссылке | Экран |
|-----------------|--------|
| `recovery` | «Новый пароль» (только «Забыли пароль?») |
| `signup` | Вход в приложение с паролем из регистрации (не смена пароля) |

Раньше любая ссылка с `access_token` ошибочно открывала смену пароля — это исправлено в `src/lib/passwordRecovery.ts`.

## 3. Отправитель «не Supabase» (рекомендуется для прод)

Стандартный SMTP Supabase ограничен и в заголовке часто видно «Supabase». Для бренда:

1. **Authentication** → **SMTP Settings** → включить **Custom SMTP**.
2. Подключить провайдера (Resend, SendGrid, Mailgun, Яндекс 360 и т.д.).
3. Указать, например:
   - **Sender email**: `noreply@ваш-домен.ru`
   - **Sender name**: `StaySee AI`

Без custom SMTP меняется только **текст** письма; домен отправителя может остаться системным.

## 4. Проверка

1. На экране входа: «Забыли пароль?» → ввести email.
2. Письмо на русском, кнопка «Задать новый пароль».
3. Клик → приложение → форма нового пароля → вход в аккаунт.

## Переменные в шаблонах

Supabase подставляет: `{{ .ConfirmationURL }}`, `{{ .Email }}`, `{{ .SiteURL }}` и др. [Документация](https://supabase.com/docs/guides/auth/auth-email-templates).

## Если «Не удалось отправить письмо»

После нажатия «Забыли пароль?» приложение показывает **точную причину** от Supabase (см. также консоль браузера F12 → `resetPassword`).

| Что проверить | Где в Dashboard |
|---------------|-----------------|
| **Custom SMTP с ошибкой** (самое частое после настройки бренда) | Authentication → **SMTP Settings** — отключите Custom SMTP и проверьте снова; или исправьте host, порт, логин, пароль API |
| **Redirect URL** | Authentication → **URL Configuration** — в списке должен быть **ровно** адрес из адресной строки браузера: `http://localhost:5173` или `http://127.0.0.1:5173` (это разные строки) |
| **Поломанный HTML шаблона** | Email Templates → Reset password — вставьте заново из `recovery.html`, не удаляйте `{{ .ConfirmationURL }}` |
| **Captcha** | Authentication → **Attack Protection** / captcha — если включена, в приложении её нет → отключите или настройте |
| **Лимит** | Подождите 10 мин, не жмите кнопку много раз подряд |

### Письмо не пришло, но ошибки нет

Supabase **не сообщает**, есть ли аккаунт с таким email (защита от перебора). Проверьте:

1. **Authentication → Users** — есть ли пользователь с этим email.
2. Папку **Спам**.
3. Что вы вводите тот же email, что при регистрации.

## «На экран входа» без смены пароля — снова кидает на смену пароля

После ссылки recovery в адресе остаётся `#access_token&type=recovery`. Если уйти на вход **без** сброса флага, при обновлении страницы приложение снова откроет «Новый пароль».

**Сейчас в приложении:** кнопка «На экран входа» сбрасывает recovery-сессию, флаг и hash в URL. Если застряли на старой версии — **«Сбросить вход»** на загрузке или очистите данные сайта для этого домена.

## Застряли после ссылки (вход не пускает, ссылка «сгорела»)

**Важно:** пока вы не нажали «Сохранить» на экране нового пароля, **старый пароль в базе не меняется**. Ссылка из письма одноразовая — повторно та же ссылка не откроется.

### Что сделать сейчас

1. **Обновите страницу приложения** (после последнего кода) — по ссылке должен открываться экран «Новый пароль», а не только вход.
2. На экране входа снова **«Забыли пароль?»** → новое письмо → откройте ссылку **в том же браузере**, где запущен StaySee (`localhost:5173` или ваш домен).
3. Перед новой ссылкой при необходимости: **«Сбросить вход»** на загрузке или очистите данные сайта для localhost.
4. В Dashboard: **Authentication → Users** → ваш email → **Send password recovery** (альтернатива п. 2).

### Для разработчика (новая ссылка из терминала)

```bash
node scripts/auth-recovery-link.mjs ваш@email.com
```

Скопируйте ссылку из вывода и откройте в браузере с запущенным `npm run dev`.

## Проверка без нового email

Новый ящик не обязателен.

| Что проверить | Как |
|---------------|-----|
| **Вход после регистрации** | Тот же email + пароль с экрана регистрации → «Войти». Если пускает — подтверждение уже сработало, смена пароля не нужна. |
| **Ссылка «Подтвердить email»** (без письма) | В `.env` нужен `SUPABASE_SERVICE_ROLE_KEY`. Удалите тестового пользователя в **Authentication → Users** (или используйте ещё не зарегистрированный email), затем: `node scripts/auth-signup-link.mjs ваш@email.com` — откройте ссылку в браузере с `npm run dev`. Должен открыться **main/онбординг**, не «Новый пароль». |
| **Ссылка сброса пароля** | `node scripts/auth-recovery-link.mjs ваш@email.com` — должен открыться **только** экран «Новый пароль». |
| **Тот же почтовый ящик, «новый» для Supabase** | У Gmail/Яндекс: `имя+staysee2@gmail.com` — письма приходят на `имя@gmail.com`. |

В Dashboard: **Users** → ваш email → можно **Confirm email** вручную, если письмо не дошло, но войти уже нужно с паролем регистрации.

### Чеклист «всё сделала»

- [ ] Reset password: HTML + тема `Сброс пароля — StaySee AI`
- [ ] Redirect URLs: `http://localhost:5173` **и** при необходимости `http://127.0.0.1:5173`
- [ ] SMTP: либо **выключен** (письма от Supabase, но текст русский), либо **включен и проверен** (тестовое письмо из провайдера)
- [ ] В приложении после запроса — зелёное сообщение, не красная ошибка
