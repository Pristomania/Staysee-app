# Триггеры «запомни» и «записки»

Клиентская логика в `src/lib/chatCaptureTriggers.ts`, UI в `ChatScreen`.

Распознавание фраз — по подстрокам и кириллическим границам слов (не `\b` из JS — с русским не работает).

## Память беседы (`conversation_summary`)

Срабатывает на реплику пользователя:

- **запомни** … / **помни что** …
- **можешь (это) запомнить?**, **запомнишь**, **не забудь** — просьба к StaySee, не записки

Если в фразе только «это» — в память попадает **последний ответ AI** (или предыдущая реплика пользователя), а не слово «это».

Факт добавляется в `important_events` (без дубликатов). Показывается короткое подтверждение «Запомнила».

## Записки себе (`progress_entries`)

**Пользователь** — записать, запиши, выписать, в записки, вернуться к этому. **Не** «можешь запомнить» (это память беседы).

**Ответ StaySee** — если в тексте есть приглашение записать (запиши, записать мысли, в записки…).

Компактная пилюля **над полем ввода** («В записки» + ✕). Не сдвигает ленту сообщений. Закрытие: ✕, новое сообщение, начало набора, таймаут ~8 с.

## Различие

| Фраза | Действие |
|-------|----------|
| «Запомни, что …» | Авто в память беседы |
| «Ты можешь это запомнить?» | Память беседы (смысл из последнего ответа) |
| «Надо записать …» | Подсказка записок |
| Модель: «можешь записать мысли» | Подсказка записок |
| Модель: «запомню» / «запомнила» | Без подсказки записок (память на сервере) |

Фоновое обновление summary от модели после чата не заменяет явное «запомни».
