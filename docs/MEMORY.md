# Память StaySee

## Принципы (жёстко)

1. **Пять чатов** — пять изолированных бесед (`MAX_ROOMS = 5` в клиенте).
2. **Память беседы** (`conversation_summary`) — только для своего `conversation_id`.
3. **Поиск по архиву** — только сообщения **текущего** чата. Другие чаты не участвуют.
4. **«Похожая тема»** — не повод искать в другом диалоге.
5. **Сквозная память** (`user_memory`) — единственный общий слой между чатами; пользователь может **выключить** в Контексте или на экране «Память», удалять и править пункты в UI.

## Переключатель сквозной памяти

- Колонка `profiles.cross_memory_enabled` (по умолчанию `true`).
- **Выкл.:** в промпт не попадает `user_memory`; после бесед не вызывается `refreshUserLifeMemory`.
- **Вкл.:** как раньше. Память **беседы** не зависит от переключателя.

## Слои в каждом запросе к AI

| Слой | Источник | Область |
|------|----------|---------|
| Хвост реплик | `messages` (последние N по тарифу) | Текущий чат |
| Память беседы | `conversation_summary` | Текущий чат |
| Архив беседы | `conversationRetrieval.ts` | Текущий чат |
| Сквозная память | `user_memory` | Пользователь |

## Архив беседы (retrieval)

**Код:** `conversationRetrieval.ts`, `messageEmbeddings.ts`, `embeddings.ts`

### Гибрид: слова + смысл (эмбеддинги)

- Таблица `message_embeddings` (миграция `015_message_embeddings.sql`).
- Векторы только для сообщений **текущего** `conversation_id`.
- Поиск: RPC `match_conversation_message_embeddings` — **без других чатов**.
- Модель: `openai/text-embedding-3-small` через OpenRouter (`OPENROUTER_API_KEY`).

### Поведение

- До ответа: до 35 сообщений без вектора индексируются; затем semantic + keyword merge.
- После ответа (фон): новые реплики пары user/ai получают эмбеддинги.
- Keyword + режим «помнишь»: подбор по словам из вопроса (любая тема, не зашитый список).
- Semantic: перефразирования в **этой же** беседе.
- Если вопрос про «вспомни» — дополнительно недавние и разбросанные фрагменты архива.

### Backfill для старых чатов

```bash
npx supabase db push
npm run backfill:embeddings
```

Опционально один чат: `npm run backfill:embeddings -- --conversationId <uuid>`

## Обновление сводки

- Каждые ~10 новых сообщений, при поправках, при пустой сводке, если сводка старше 24 ч.
- Eager refresh перед ответом, если сводка устарела или пустая.
- **Код:** `summaryRefresh.ts`, `memory.ts`

## Аналитика

Токены архива учитываются в `summary_tokens` вместе с блоком памяти беседы. См. `docs/ANALYTICS.md`.
