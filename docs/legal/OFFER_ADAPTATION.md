# Адаптация оферты GPTs → StaySee (приложение)

Чеклист для правки юридически согласованного текста под веб-приложение **StaySee AI**. Отметьте каждый пункт после замены в `offer.md` / `offer.ts`.

## 1. Стороны и продукт

| Было (типично для GPTs) | Стало (StaySee App) |
|-------------------------|---------------------|
| Название бота / Custom GPT / «ассистент в ChatGPT» | **StaySee AI** — веб-приложение (сайт / PWA), не продукт OpenAI |
| Платформа OpenAI / ChatGPT | **Сервис StaySee** на собственном домене; ИИ через API (OpenRouter и др.) — **без** утверждения, что OpenAI — исполнитель |
| «Пользователь ChatGPT» | **Пользователь** / **Зарегистрированный пользователь** приложения StaySee |

## 2. Предмет договора

- Уточнить: доступ к **личному кабинету**, **беседам**, **памяти бесед**, **запискам**, лимитам сообщений по тарифу.
- Убрать упоминания магазина GPTs, подписки ChatGPT Plus (если были), если оплата идёт **напрямую** StaySee.
- Добавить (если нет): сервис **не** медицинский, **не** психотерапия, **не** экстренная помощь — см. дисклеймер в приложении.

## 3. Регистрация и акцепт

- Акцепт: регистрация + галочка **«Подтверждаю, что мне исполнилось 18 лет»** + «Принимаю публичную оферту» + начало использования (см. `RegisterScreen`, `ROOM_COPY`).
- Email и пароль — учётная запись Supabase Auth.
- В оферте явно: сервис только для лиц **18+**; без подтверждения возраста регистрация недоступна.

## 4. Тарифы и оплата

Согласовать с `user_usage_tiers` и `TIER_CONFIG`:

| Тариф | Сообщений в день (сейчас в коде) |
|-------|----------------------------------|
| Free | 50 |
| Basic | 200 |
| Premium | 1000 |

- Прописать: лимит **суточный**, сброс раз в 24 ч.
- Цены в рублях / валюта — **ваши** из коммерческого плана (в оферте — конкретные суммы или «по прайсу на сайте»).
- Возвраты, пробный период, автопродление — перенести из GPTs-оферты или согласовать с бухгалтерией.
- До запуска оплаты: формулировка «бесплатный тариф» + «платные тарифы подключаются отдельно».

## 5. Данные и конфиденциальность

- Ссылка на **Политику конфиденциальности** (экран в приложении / отдельный URL на VPS).
- Хранение: беседы, email, память бесед, сквозная память — в Supabase, RLS.
- Удаление бесед и памяти — действия пользователя в приложении.
- Не использовать переписку для обучения моделей (если так в продукте).
- Субобработчики: хостинг (Vercel/VPS), Supabase, провайдер API ИИ — перечислить как в исходной оферте.

## 6. Ограничение ответственности

- Сохранить блоки про ИИ, ошибки, «галлюцинации», отсутствие гарантии результата.
- Ответственность за решения — на пользователе (как в дисклеймере).

## 7. Расторжение и блокировка

- Удаление аккаунта (когда появится в UI) / нарушение правил / `is_suspended` при злоупотреблении API.

## 8. Реквизиты и контакты

- ИП / ООО, ИНН, ОГРН, адрес, email поддержки — **актуальные** для StaySee (не для GPTs-витрины).

## 9. Версионирование

- В `OFFER_META.version` и `effectiveDate` — дата публикации в приложении.
- При смене оферты: уведомление в приложении / email (если обязаны по закону).
