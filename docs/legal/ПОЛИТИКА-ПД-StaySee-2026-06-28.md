## 2. Принципы обработки персональных данных

Оператор обрабатывает персональные данные на основании следующих принципов (статья 5 152-ФЗ):

- **Законность и справедливость** — обработка осуществляется на законных основаниях, с соблюдением прав субъектов.
- **Ограничение цели** — данные собираются для конкретных, заранее определённых целей и не обрабатываются иным образом, несовместимым с этими целями.
- **Минимальность** — обрабатываются только те данные, которые необходимы для достижения заявленных целей.
- **Точность** — Оператор принимает меры по поддержанию актуальности данных.
- **Ограничение хранения** — данные не хранятся дольше, чем это необходимо для достижения целей обработки.
- **Целостность и конфиденциальность** — Оператор принимает технические и организационные меры для защиты данных от несанкционированного доступа.

### 5.3. Робокасса (ООО «Робо.Ру», Россия)
- **Роль:** Платёжный шлюз для приёма оплаты подписки.
- **Передаваемые данные:** Данные банковской карты Пользователя (обрабатываются Робокассой напрямую, Оператор данные карты не получает), сумма платежа, идентификатор заказа.

- **Роль:** Распространение мобильного приложения через App Store и Google Play.

## 8. Меры по обеспечению безопасности

Оператор применяет следующие технические и организационные меры защиты персональных данных:

- **Шифрование при передаче:** все соединения защищены по протоколу TLS 1.2 и выше.
- **Шифрование в хранилище:** данные хранятся в зашифрованном виде на серверах Supabase.
- **Аутентификация:** защита учётных записей паролями, хранящимися в виде хеша (bcrypt).
- **Ограничение доступа:** доступ к данным Пользователей в базе данных разграничён на уровне Row Level Security (RLS) — Пользователь видит только свои данные.
- **Мониторинг:** ведение технических логов для выявления несанкционированного доступа.
- **Минимизация передаваемых данных:** при каждом обращении к языковой модели передаётся только контекст, необходимый для формирования ответа.
